# Feedback Loop Protocol

Use this when the user asks for continued iteration, repeated improvement, benchmark comparison, or skill-first refinement.

## Core Rule

The loop is not "make it prettier." The loop is: generate or inspect, score, identify the weakest high-impact category, improve one focused layer, verify, then update the skill if the lesson generalizes.

## Loop Order

1. **Choose target**: skill rule, template deck, website sample, gallery page, or public deployment.
2. **Gather references**: use `DESIGN_REFERENCE_CASES.md` and current industry examples when the topic is likely to have evolved.
3. **Audit**: use `DESIGN_JUDGMENT.md`.
4. **Score**: use `DESIGN_SCORECARD.md`, including hard caps.
5. **Prioritize**: choose the lowest high-impact category, not the easiest cosmetic change.
6. **Improve**: make one focused change round.
7. **Verify**: use `DESIGN_VERIFICATION.md`.
8. **Rescore**: update only the changed categories and any hard cap.
9. **Learn**: if the lesson applies beyond one sample, update the relevant skill file.
10. **Continue or switch**: continue until remaining gains require new real content/assets, or switch to the next archetype.

## Skill-First Mode

When the user says the focus is upgrading the skill:

- Prefer adding or tightening reusable rules over polishing one sample.
- Use a sample only to pressure-test the rule.
- Avoid adding rules that are just one-off preferences.
- Promote repeated findings into `WEBSITE_GRADE.md`, `TEMPLATE_GRADE.md`, `COLOR_STRATEGY.md`, `DESIGN_JUDGMENT.md`, `DESIGN_SCORECARD.md`, `DESIGN_VERIFICATION.md`, or `DESIGN_REFERENCE_CASES.md`.
- Keep rules actionable: a future agent should know what to inspect and what to change.

## Iteration Log

Keep a compact note during long loops:

```text
Target:
Reference pattern:
Before score:
Weakest category:
Change:
Verification:
After score:
Skill update:
Next target:
```

Do not preserve long narrative logs unless the user asks. Keep durable lessons in the skill files.

## Stop Conditions

Stop or switch target when:

- Score reaches 88+ and remaining gains are taste-level refinements.
- Score reaches 95+ for a flagship sample.
- The lowest remaining issue requires real brand assets, real product screenshots, real customer proof, or user-provided content.
- The current archetype has hit a bottleneck and another archetype has larger structural gaps.
- The skill rule already covers the lesson and another repetition would not improve future output.

## Common Loop Mistakes

- Polishing color before fixing information architecture.
- Adding a new section instead of making the existing decision path clearer.
- Comparing only to visually impressive sites, not task-effective ones.
- Updating a sample but not updating the skill with the generalized lesson.
- Updating the skill with vague advice that cannot be scored or verified.
